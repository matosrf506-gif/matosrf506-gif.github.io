## 一、Meta 风控的核心逻辑：账号信任评分

Meta 对每一个广告账号和 Business Manager 都维护着一套隐性的信任评分系统。这套系统会持续分析账号的各类行为信号，评分过低时触发限制措施，评分极低时直接封禁。

影响信任评分的因素包括：

| 风险因素 | 风险等级 | 说明 |
|---------|---------|------|
| 使用数据中心 IP 登录 | 极高 | 系统直接识别为自动化行为或代理欺诈 |
| IP 归属地与账号注册地不符 | 高 | 触发地理位置异常警报 |
| IP 被多个广告账号共用 | 高 | 关联账号检测，一个被封则连累其他 |
| IP 有历史违规记录 | 高 | 共享 IP 池的污染问题 |
| 频繁切换 IP | 中高 | 不符合正常用户行为模式 |
| 广告素材质量问题 | 中 | 触发内容审核，但影响权重低于账号信任 |
| 支付信息异常 | 高 | 信用卡被拒或多账号绑定同一支付方式 |
| 登录设备频繁更换 | 中 | 与正常商业用户行为不符 |

从上表可以看出，**IP 相关因素在风控体系中占据了相当大的权重**。很多广告主花大量时间打磨素材、研究政策，却忽视了每次登录账号时使用的 IP 环境。

## 二、为什么机房 IP 和共享代理是定时炸弹？

许多广告主在国内操作海外 Facebook 账号时，会使用各类代理服务。但代理的种类直接决定了风险等级：

### 机房 IP（数据中心代理）
Meta 的数据库中存储了全球主要云服务商和 IDC 的 IP 段信息。当你通过 AWS、阿里云或普通 VPS 的 IP 访问 Facebook，系统几乎可以确定这是一个非真实用户行为——真实的美国小企业主不会通过 AWS 的 IP 登录自己的广告账户。这类访问会立即触发额外的安全验证，长期使用会显著拉低账号信任分。

### 共享住宅 IP 池
这类代理服务将大量住宅 IP 放入共享池中，用户随机分配。问题在于，同一个 IP 可能被数百个用户在同一天使用，其中包括有违规历史的账号。Meta 会追踪 IP 的历史行为，一个"脏" IP 会污染你的账号信任记录。

更严重的是，如果某个共用你 IP 段的账号因为投放违规广告被封，Meta 可能通过 IP 关联，将风险传导到你的账号。

## 四、BM 和广告账号的 IP 管理实操指南

### 4.1 一个 BM 对应一个固定住宅 IP

最佳实践是为每个 Business Manager 分配一个专属的 NasaVPN 节点，该节点固定不换，所有与该 BM 相关的操作（账号创建、广告投放、支付操作）均通过同一个 IP 进行。

### 4.2 多个 BM 的隔离配置

如果你管理多个 BM（例如不同品牌或不同客户），建议：

1. 为每个 BM 分配独立的 NasaVPN 节点（不同地区的住宅 IP）
2. 使用独立的浏览器 Profile（AdsPower、Multilogin 等反检测浏览器）
3. 使用不同的 Facebook 个人账号作为管理员
4. 确保不同 BM 之间无交叉操作

### 4.3 素材审核期间的 IP 稳定性

广告素材进入审核队列时，Meta 会重新评估账号的可信度。在此期间保持 IP 稳定（不切换节点）可以避免额外触发安全验证，有助于审核顺利通过。

### 4.4 账号申诉期间的操作建议

当账号已经被限制、正在申诉阶段时：

- **保持使用同一个固定住宅 IP**，不要因为紧张而频繁切换
- 通过 IP 核验证明你的账号来自固定地点操作，有助于申诉成功
- 避免在申诉期间进行大量新的广告创建操作
