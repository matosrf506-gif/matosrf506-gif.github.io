Claude Code 是目前最强的 AI 编程助手之一，但它有个致命弱点：**极其依赖网络稳定性**。

你写了半小时代码，Claude 正在生成一大段逻辑，突然：连接中断。整个上下文丢失，重新来过。这种体验让人抓狂。

问题不在 Claude，在你的网络节点。

---

## 一、Claude Code 为什么特别容易断线

普通对话只需要短暂的请求-响应，对网络要求不高。但 Claude Code 不一样：

- **超长上下文**：一次会话可能传输几万 token 的代码和上下文
- **长时生成**：生成复杂代码片段可能需要 10-30 秒持续输出
- **多轮迭代**：调试、修改、再生成，整个会话可能持续数小时
- **流式输出**：Claude 的 stream 模式对长连接稳定性要求极高

这意味着哪怕 1-2% 的丢包率，都可能在几十分钟的会话里累积成一次断线。

**普通 VPN 的共享节点高峰期丢包率轻松超过 5%，还有 200ms+ 的延迟抖动。** 这就是为什么你用普通 VPN 时 Claude Code 总是断。
